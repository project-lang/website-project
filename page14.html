Describe which are the appropriate filetypes for websites

Appropriate file types for websites depend on the type size and loading speed. Here are some of the better examples: Graphics:
JPEG (Joint Photographic Experts Group) = Usually the most compact and quickest to load, although there could be some loss in quality.
TIFF (Tagged Image File Format) = Good quality but slower to load as it is large.
PNG (Portable Network Graphics) = With these types of images it is possible to be more creative than with a JPEG. e.g. PNG supports transparency, sometimes necessary when the background needs to be removed without messing up the picture. Better quality but the file is larger than a JPEG.
GIF (Graphics Interchange Format) = These types of files are used to play animated graphics, of short duration.
Sound:
MP3 = This is the usual file type for listening to audio. Good quality and highly compressed.
WAV = Not such a good choice for audio, best for short duration.
Sound and Vision:
MP4 or MPEG (Moving Picture Experts Group ) = These files are good for audio and video, still images, text. Becoming the preferred choice amongst many users.
AVI (Audio/Video Interleaved) = File has to be completely downloaded before it can be viewed.

HTML (Hyper Text Markup Language) = These are basic commands for composing the skeleton of a website.
DOC (Microsoft Word Document) = Probably the most commonly used word processing software in the world.
TXT (Text) = Basically it is just a file composed of text.
EXE (Executable File) = Usually what downloaded files come as.
PDF (Portable File Document) = These were created in order to provide uniformity across the Internet for documents.
ZIP (Zipped File) = Groups of downloadable files are often composed of zip files. You need an extractor to execute them.
